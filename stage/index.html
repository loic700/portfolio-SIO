Update Manager : Orchestration de Mises à Jour

×

Pourquoi ?

Éviter les mises à jour "aveugles" qui peuvent casser des services critiques. Ce script permet de choisir précisément quel paquet mettre à jour sur chaque machine (PVE ou VM) via une interface visuelle.

Fonctionnement

Le système repose sur un cycle automatisé entre Ansible et Python :

  • Scan : Un playbook Ansible récupère les paquets non à jour et génère des fichiers JSON.
  • Interface : Le script affiche un tableau de bord dynamique pour la sélection des paquets.
  • Interface & Ergonomie : Ajout de boutons de contrôle (Tout Cocher, Tout Décocher, Ignorer) et amélioration du CSS pour un accès plus simple et intuitif aux informations.
  • Validation : La sélection crée un fichier selection.json utilisé par Ansible pour l'installation.

Documentation

Une documentation simple et précise accompagne ce script pour une prise en main rapide, de la phase de scan à la validation finale.

Upgrade OS : Migration de Distribution Automatisée

×

Pourquoi ?

Réaliser des montées de version majeures (Dist-Upgrade) sur un parc de serveurs est une opération critique et chronophage. Ce script sécurise le processus en permettant de piloter la migration de chaque machine depuis une interface visuelle.

Fonctionnement

Le livre de jeu (Playbook) orchestre la migration en plusieurs phases :

  • Scan Initial : Récupération automatique des distributions actuelles (IP, Version, Nom de code) sur tout le parc.
  • Interface & Ergonomie : Utilisation d'un script web pour sélectionner facilement la version cible pour chaque VM.
  • Migration forcée : Automatisation du changement des dépôts (sources.list), gestion des conflits de configuration (force-confold) et redémarrage automatique après succès.

Documentation

Une machine virtuelle a été créée pour permettre le test du script sans impacter la VM de production. Il n’est donc pas nécessaire de masquer l’adresse IP.

Augmentation de Stockage (15 Go vers 150 Go)

×

Pourquoi ?

Cette opération a été effectuée sur différentes machines virtuelles (VMs) pour augmenter leur capacité de stockage sans interruption de service, répondant ainsi aux besoins croissants de données.

Comment ?

L'extension a été réalisée à chaud en suivant ces étapes techniques :

  • Identification : lsblk
  • Extension de la partition : sudo growpart /dev/sda 3
  • Redimensionnement du volume physique : sudo pvresize /dev/sda3
  • Extension du volume logique : sudo lvextend -l +100%FREE /dev/mapper/ubuntu--vg-ubuntu--lv
  • Mise à jour du système de fichiers : sudo resize2fs /dev/mapper/ubuntu--vg-ubuntu--lv
